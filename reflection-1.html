In a couple of sentences, how would you describe the command line in plain English? Can you think of an analogy for it?

What's the difference between git and GitHub?
- Git is a version control system that helps to track files so you don’t override/ clash changes while working with multiple teammates. If we do override the changes, we get notified to merge, etc in an efficient manner. When working in a team, Git is installed in each of our local machines. Now, this is where Git Hub can come into the picture. Git Hub is the cloud where you can sync these files from our local machines with your teammates, it’s a central source/ server/ service.

Can you think of an analogy to describe them?
- An analogy to describe them is: Let’s say, me and my Graphic Designer friend are collaborating on a brief. We like to work from home and at our own time. We are in the final draft of their project and each have multiple versions of ‘’project’, prject-final’, ‘finalllll I swear this time’ and this is where Git comes in. Git helps track these files so I know the changes my friend is making based on our feedback and I am alerted if I am overriding their changes. So this, is Git. Git hub, is separate, it’s like behind the scenes/ cloud server. We can sync our files onto this cloud system. Git is local, whereas, Git Hub is online.

What is main?
The main is the code that runs smoothly and it's the ideal version of the code.

Why create a Branch?
Branches are important when let's say, teammates, want to make changes. They can create a branch and make changes and not have it affect the main. Once approved and tested, it can be merged, but until then, when it's work in progress, branch is the place to make the tweaks!

How would you describe stage and commit to your non-tech-savvy friend?
An analogy I would use to describe Stage and commit to my non-tech-savvy friend is that of a relay race. I would explain from the Working state.Think of the time when you may have participated/ watched a relay race. Imagine yourself being in one of the teams. The race is about to begin, and you are 2nd in line. Great, now hit pause to that scenario. The person in front of you, is preparing his position and making changes to his posture to be best prepared to run when the whistle goes off. This friend, the first person is like a working state. A working state is that in which changes have been made, but no committing has been done, I.e you have not started running yet. Now, the whistle goes off. Your friend runs and you are next in line to go. You are all set and ready and it is your chance next. You are in a Staged state. Here, you have been marked, it is known, that you will running next in the relay, and in this scenario, ‘the next commit snapshot’. Once you reach the person you have to give the baton to, you have done your part. Your speed time is marked and your part of the race is complete. This state is called the Committed state. Here, the baton is safely delivered to the next team player, the ‘data is safely stored in the local database’.
